> 最后更新：2026-05-26

## 一、项目概述

### 目标
自动爬取西南财经大学各社团微信公众号的活动信息，提供统一的浏览、搜索、筛选界面，方便学生一站式查看全校社团活动。

### 架构
```
读取: 用户 → GitHub Pages ← site/data/*.json (静态数据)
                           ↑
                     merge_data.py  (数据合并去重)
                          ↑
            ┌───────────┴───────────┐
      extract_activity.py    manual_activities.json
            ↑                   (人工提交通道)
     crawl_wechat.py                 ↑
            ↑                  Cloudflare Worker
     微信公众号 API          (publish-worker.js)
     (cgi-bin/appmsgpublish)      ↑
                             用户浏览器 (发布表单)
```

### 技术栈
| 层 | 技术 |
|------|------|
| 前端 | 原生 HTML + CSS + JS (自研主题) |
| 二维码 | qrcode-generator (表格渲染) |
| 截图 | html2canvas (自托管) |
| 爬虫 | Python + requests / httpx / Camoufox |
| 发布 API | Cloudflare Worker (JS, 中国可达) |
| CI/CD | GitHub Actions (每天 UTC 18:00) |
| 托管 | GitHub Pages (主要) + Vercel (历史遗留) |
| 数据格式 | JSON 静态文件 |

## 六、已知问题

### 6.1 严重问题

1. **活动状态不准确**
   - 部分历史活动因 `start_time` 含年份混乱（如提取到 2023 年但写成了 2026/2027）被标记为 `upcoming`/`ongoing`
   - **根源**: 爬虫只获取了文章摘要而非全文，提取规则无法从摘要中准确提取时间
   - **当前**: 通过标题关键词回退和发布时间启发式已大幅改善（ended 从 7→52），但仍有个别残留

2. **提取率低**
   - 大量活动缺少地点和联系方式（~50%+）
   - **根源**: `WECHAT_COOKIE` 有效性影响正文抓取；Camoufox 反检测浏览器依赖系统安装
   - **潜在方案**: 配置 `DEEPSEEK_API_KEY` 运行 `--enrich` AI 补全缺失字段

### 6.2 代码质量问题

3. **`all_articles` 比较问题**
   - `crawl_wechat.py` 使用 `not in` 比较字典，在 Python 中 `not in` 使用 `==` 比较，对于新创建的字典对象不可靠
   - **建议**: 改用文章 URL 或 aid 作为去重键

4. **`extract_activity.py` — BEIJING_TZ 引用顺序**
   - `compute_status()` 函数引用了 `BEIJING_TZ`，但变量定义在函数之后，可读性差

5. **无单元测试**
   - 仅 `scripts/tests/test_wechat_to_md.py` 一个测试文件
   - 提取规则变更难以验证

6. **日志不统一**
   - 部分输出用中文，部分用英文
   - 无日志级别控制

### 6.3 运维问题

7. **`WECHAT_COOKIE` 过期静默**
   - cookie 过期后爬虫跳过，不报警
   - 无通知机制

8. **数据量增长**
   - 每天 UTC 18:00 全量爬取，随着社团增加请求量线性增长
   - 当前 27 个公众号 × 20 篇文章 × 10s 延迟 ≈ 5400s ≈ 90 分钟，可接受
   - 扩展到 200+ 社团后需考虑优化策略
